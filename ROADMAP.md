# User Research
 Determine who we are building for
 - Who is the target user?
 - What are their main goals (relating to the problem we are trying to solve)?
 - What are the biggest pain points they experience when trying to achieve their goals?

# Feature definition 
Describe the most important features of our product
- The feature list has the following info: 
```
  [name, description, purpose, dependencies, tech requirements, "stakeholder" feedback]
```
here is a simple feature:
```
  ["simulate drift",
   "the program should be able to simulate how genetic drift works based on the input",
   "users could test how genetic drift change with population size",
   ["simulate generations"],
   "multiple locus and genes",
   none
  ]
```

# Roadmap
The goal of this document is to align the project resources around goals for the product, and describes the priority order/focus for completion and how each feature relates to the others. You created a list of features you need to deliver in the previous phase, and now you will plan how they can be completed in a coherent manner.
